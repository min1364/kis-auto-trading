# KIS Open API Mock Auto Trading System

## 1. 프로젝트 개요

본 프로젝트는 한국투자증권 KIS Developers Open API를 사용하여 국내주식 모의투자 환경에서 자동매매 시스템을 구현한 프로젝트이다.

자동매매 대상 종목은 SK하이닉스(000660)이며, REST API를 사용하여 현재가 조회, 계좌 잔고 조회, 보유 여부 판단, 매수 주문, 매도 주문, HOLD 판단, 반복 실행, 거래 기록 저장 기능을 구현하였다.

본 프로젝트는 실전투자 환경이 아닌 한국투자증권 모의투자 환경에서만 실행되도록 구성하였다.

---

## 2. 개발 목적

본 프로젝트의 목적은 단순히 주가를 조회하는 프로그램을 만드는 것이 아니라, 실제 증권사 API를 활용하여 자동매매 시스템의 기본 구조를 구현하는 것이다.

이를 위해 다음 기능을 포함하였다.

* API 인증
* access token 발급 및 재사용
* 국내주식 현재가 조회
* 모의투자 계좌 잔고 조회
* 보유 여부 확인
* 평균 매입가 기반 수익률 계산
* BUY / SELL / HOLD 자동 판단
* 모의투자 주문 실행
* 일정 시간 동안 반복 실행
* 거래 기록 CSV 저장
* API 호출 제한 대응

## 5. 환경변수 설정

보안상 APP Key, APP Secret, 계좌번호는 코드에 직접 작성하지 않고 `.env` 파일에서 불러오도록 구현하였다.

`.env` 예시는 다음과 같다.

```env
KIS_ENV=mock

KIS_APP_KEY=your_mock_app_key
KIS_APP_SECRET=your_mock_app_secret

KIS_ACCOUNT_NO=your_mock_account_number
KIS_ACCOUNT_PRODUCT_CODE=01

STOCK_CODE=000660
ORDER_QTY=1

DRY_RUN=true

TAKE_PROFIT_PCT=1.0
STOP_LOSS_PCT=-1.0

RUN_INTERVAL_SECONDS=120
RUN_DURATION_MINUTES=60
```

실제 `.env` 파일에는 개인 API 키와 계좌번호가 포함되므로 GitHub에 업로드하지 않는다.

## 7. 자동매매 전략

본 프로젝트는 보유 여부와 수익률을 기준으로 하는 단순 규칙 기반 자동매매 전략을 사용한다.

전략 흐름은 다음과 같다.

1. 한국투자증권 Open API를 통해 access token을 발급받는다.
2. 국내주식 현재가 조회 API로 대상 종목의 현재가를 조회한다.
3. 모의투자 계좌의 잔고를 조회하여 대상 종목의 보유 수량과 평균 매입가를 확인한다.
4. 대상 종목을 보유하고 있지 않으면 1주 시장가 매수 주문을 실행한다.
5. 대상 종목을 보유하고 있으면 현재가와 평균 매입가를 비교하여 수익률을 계산한다.
6. 수익률이 +1% 이상이면 익절을 위해 1주 시장가 매도 주문을 실행한다.
7. 수익률이 -1% 이하이면 손절을 위해 1주 시장가 매도 주문을 실행한다.
8. 수익률이 -1% 초과, +1% 미만이면 추가 주문 없이 HOLD한다.

따라서 본 자동매매 시스템은 단순 매수 프로그램이 아니라, 현재가 조회, 잔고 조회, 보유 여부 판단, 수익률 계산, 매수, 매도, HOLD 판단을 포함한 자동매매 구조를 갖는다.

---

## 8. 매매 판단 기준

```text
미보유 상태 → BUY
보유 중 & 수익률 >= +1% → SELL
보유 중 & 수익률 <= -1% → SELL
보유 중 & -1% < 수익률 < +1% → HOLD
```

수익률 계산식은 다음과 같다.

```text
수익률(%) = (현재가 - 평균매입가) / 평균매입가 × 100
```

## 9. DRY_RUN 모드

본 프로젝트는 안전한 테스트를 위해 `DRY_RUN` 모드를 포함한다.

```env
DRY_RUN=true
```

일 때는 실제 주문 API를 호출하지 않고, 매매 판단 결과만 CSV에 기록한다.

```env
DRY_RUN=false
```

일 때는 모의투자 환경에서 실제 주문 API를 호출한다.

이를 통해 실수로 주문이 발생하는 것을 방지하면서 API 연결, 현재가 조회, 잔고 조회, 매매 판단 로직을 먼저 검증할 수 있다.

## 10. 반복 실행 기능

초기 버전은 `main.py`를 실행하면 한 번만 현재가 조회, 잔고 조회, 매매 판단을 수행하는 구조였다. 이후 자동매매 시스템답게 일정 시간 동안 반복 실행할 수 있도록 `bot_runner.py`를 추가하였다.

`bot_runner.py`는 `.env`의 설정값을 이용하여 지정된 간격마다 `main.py`의 자동매매 로직을 반복 실행한다.

```env
RUN_INTERVAL_SECONDS=120
RUN_DURATION_MINUTES=60
```

위 설정은 120초 간격으로 60분 동안 자동매매 판단을 반복 수행한다.

실행 명령어는 다음과 같다.

```bash
python bot_runner.py
```

이를 통해 1회성 실행이 아니라, 장중 일정 시간 동안 자동매매 판단을 반복하도록 구성하였다.

## 11. API 호출 제한 처리

반복 실행 중 한국투자증권 Open API의 초당 거래건수 제한 오류가 발생할 수 있다. 실제 테스트 과정에서도 짧은 시간 안에 현재가 조회와 잔고 조회 API가 연속 호출되면서 초당 거래건수 제한 응답을 확인하였다.

이를 해결하기 위해 현재가 조회, 잔고 조회, 주문 요청 사이에 `time.sleep()`을 추가하여 API 호출 간격을 조절하였다.

또한 토큰 발급 API는 짧은 시간 내 반복 호출 시 제한될 수 있으므로, 한 번 발급받은 access token을 `logs/token.json`에 저장하고 재사용하도록 구현하였다.

이를 통해 실제 증권사 API를 사용하는 자동매매 시스템에서는 단순히 주문 기능을 구현하는 것뿐만 아니라, 토큰 발급 제한과 초당 API 호출 제한을 고려해야 한다는 점을 반영하였다.

## 12. 실행 방법

### 1. 패키지 설치

```bash
pip install -r requirements.txt
```

또는 Windows 가상환경을 사용하는 경우 다음 명령어를 사용할 수 있다.

```bash
python -m pip install -r requirements.txt
```

### 2. `.env` 파일 생성

프로젝트 폴더에 `.env` 파일을 만들고 모의투자용 APP Key, APP Secret, 모의투자 계좌번호를 입력한다.

### 3. 안전 테스트 실행

처음에는 반드시 다음과 같이 설정한다.

```env
DRY_RUN=true
```

그 후 실행한다.

```bash
python main.py
```

### 4. 모의투자 주문 실행

현재가 조회와 잔고 조회가 정상적으로 실행되는 것을 확인한 뒤, 다음과 같이 변경한다.

```env
DRY_RUN=false
```

다시 실행한다.

```bash
python main.py
```

### 5. 반복 실행

일정 시간 동안 자동매매 판단을 반복하려면 다음 명령어를 실행한다.

```bash
python bot_runner.py
```

## 13. 실행 결과

`DRY_RUN=true` 상태에서 다음 기능이 정상적으로 실행되는 것을 확인하였다.

* access token 발급
* 현재가 조회
* 잔고 조회
* 보유 수량 확인
* BUY / SELL / HOLD 판단
* 거래 기록 CSV 저장

이후 `DRY_RUN=false` 상태에서 실제 모의투자 주문 API 호출을 시도하였다. 제출 당일 실행 시점이 장 종료 이후였기 때문에 주문 가능 시간이 아니라는 응답을 확인하였다.

또한 반복 실행 기능을 테스트하는 과정에서 초당 거래건수 제한 응답을 확인하였고, 이를 반영하여 API 호출 사이에 지연 시간을 추가하였다.

이를 통해 다음 사항을 확인하였다.

* 한국투자증권 Open API 인증 연결
* 모의투자 access token 발급 및 재사용
* 국내주식 현재가 조회
* 모의투자 계좌 잔고 조회
* 보유 여부에 따른 매매 판단
* BUY / SELL / HOLD 전략 로직 구현
* 주문 가능 시간 외 API 응답 확인
* 초당 거래건수 제한 응답 확인 및 호출 간격 조절
* 거래 기록 저장 기능 구현
* 일정 간격 반복 실행 기능 구현

장중 시간에 재실행하면 실제 모의투자 매수 또는 매도 주문 및 체결 기록을 확인할 수 있다.

## 15. 구현상 특징

본 프로젝트에서 구현한 핵심 특징은 다음과 같다.

### 1. REST API 기반 구현

웹소켓을 사용하지 않고 REST API만 사용하여 구현하였다.

### 2. 토큰 재사용

토큰 발급 API 호출 제한을 고려하여, 발급된 access token을 파일에 저장하고 재사용하도록 구현하였다.

### 3. 환경변수 사용

APP Key, APP Secret, 계좌번호를 코드에 직접 작성하지 않고 `.env` 파일로 분리하였다.

### 4. 매수와 매도 모두 구현

`kis_api.py`의 주문 함수는 `side="buy"`와 `side="sell"`을 모두 지원한다.

### 5. 반복 실행 기능 추가

`bot_runner.py`를 통해 자동매매 로직을 일정 간격으로 반복 실행할 수 있도록 구현하였다.

### 6. API 호출 제한 대응

API 호출 사이에 지연 시간을 두어 초당 거래건수 제한 오류를 줄이도록 개선하였다.

### 7. 단순하지만 완전한 자동매매 흐름

본 프로젝트는 복잡한 투자전략보다 자동매매 시스템의 기본 구성요소를 구현하는 데 초점을 두었다.

자동매매의 핵심 흐름인 인증, 시세 조회, 잔고 조회, 매매 판단, 주문 실행, 반복 실행, 기록 저장을 모두 포함한다.
